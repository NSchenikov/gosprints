Спринт 8: Дополнительный

Задание: Реализация обработки сущностей (задач) с использованием статусов, state machine и нотификаций
Описание

Вам необходимо доработать сервис регистрации задач, добавив процесс обработки задач через state machine и работу с асинхронными задачами. В системе есть пользователи, которые создают задачи, и даннык задачи проходят несколько этапов обработки.
Бизнес-логика

    Задача создается в статусе NEW (новая).
    Задача должна пройти два этапа валидации:
        Первый этап (VALIDATION_1) - проверяется корректность текста задачи (не пустой, не слишком длинный).
        Второй этап (VALIDATION_2) - проверяется возможность обработки задачи (например, есть ли свободный воркер, не превышен ли лимит задач пользователя и т. д.).
    Если задача прошла первый этап, но не прошла второй (например, из-за ошибки или таймаута), она должна оставаться в статусе WAITING_FOR_VALIDATION_2 до тех пор, пока его не обработает фоновый процесс.
    Фоновая горутина (state machine) каждые 30 секунд проверяет статусы задач:
        Если задача находится в WAITING_FOR_VALIDATION_2 дольше 60 минут - она автоматически удаляется.
        Если задача не может пройти VALIDATION_2, увеличивается счетчик attempts.
        Если attempts >= 3, задача переводится в статус FAILED, и пользователю отправляется нотификация.
        Если задача проходит этап VALIDATION_2, она становится READY_FOR_CLOSURE.
    Задачи в статусе READY_FOR_CLOSURE можно закрыть вручную или автоматически. После закрытия задача получает статус CLOSED.
    Закрытые задачи не обрабатываются повторно.

Требования к реализации
API для работы с задачами

    Создать задачу (POST /tasks) - принимает текст задачи и создает запись в БД.
    Получить задачу по ID (GET /tasks/{id}) - возвращает информацию о конкретной задаче.
    Обновить задачу (PUT /tasks/{id}) - обновляет статус и попытки обработки.
    Закрыть задачу (POST /tasks/{id}/close) - переводит задачу в CLOSED.
    Получить список задач по конкретному пользователю (GET /users/{user_id}/tasks) - фильтрует задачи по пользователю.

Фоновая обработка задач (State Machine)

    Использовать горутину, которая с тикером каждые 30 секунд проверяет задачи.
    Реализовать state machine, которая:
        Обновляет статус задач по правилам.
        Удаляет задачи, если они зависли больше чем на 60 минут.
        Учитывает attempts и переводит задачу в FAILED, если попыток больше 3.
